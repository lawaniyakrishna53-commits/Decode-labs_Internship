# 1. Objective (Goal)
​The primary objective of this project is to build a foundational Rule-Based AI Chatbot that can simulate basic human interaction by responding to predefined user inputs.   
​# 2. Technology 
​Programming Logic: Explicit conditional statements (if-else branching).  
​Control Flow: Continuous execution loops (e.g., while loops) to keep the chat active.  
​String Manipulation Methods: Lowercasing (.lower()) and whitespace trimming (.strip()) for text normalization.  
​Architecture Framework: The IPO Model (Input-Process-Output) which serves as a blueprint for deterministic "White Box" AI systems.
# Input & Sanitization (IPO: Input)
Raw Data Feed: Capture the user's raw string input using the program's input reader.  
Normalization Process: Clean the text by converting it to lowercase and stripping extra spaces (e.g., transforming "HeLLo " into "hello") to prevent minor user formatting variations from breaking the rules.  
# Intent Matching & State (IPO: Process)
1 . The Logic Skeleton: Use structured logic gates to pass the cleaned input through explicit if-else statements.  
Matching: Check the input against specific predefined keywords or conditional triggers (such as detecting common greetings or an exit phrase).  
# Generation (IPO: Output)
zero-hallucination text response based on whichever logical condition evaluates to true.  
Continuous Cycle: Return the flow back to Phase 1 until an explicit exit command breaks the execution loop.  
# 4. Key Results & Takeaways
successfully build a "White Box" system characterized by absolute traceability with zero risk of AI hallucinations.  
Safety Control Layer: This project acts as the basic control framework used in modern AI guardrails (like NVIDIA NeMo or Llama Guard) to filter, redact, or block unpredictable outputs generated by Large Language Models.
